# Регламент работы с пулом задач

Статус: обязательный порядок исполнения задач из рабочего пула

## Основной принцип

Задачи из пула выполняются строго последовательно, по одной за раз. Новая задача не берётся в работу, пока не завершена проверка предыдущей.

## Порядок выполнения

1. Берётся первая незавершённая задача из пула.
2. Выполняются только изменения, которые относятся к этой задаче.
3. После реализации проводится промежуточная проверка результата.
4. Если проверка пройдена, задача считается закрытой и автоматически берётся следующая задача из пула.
5. Если проверка не пройдена, переход к следующей задаче запрещён.

## Требования к промежуточной проверке

Проверка должна включать:

- проверку соответствия критериям приёмки задачи;
- проверку затронутых зон изменений;
- проверку на появление побочных проблем или регрессий;
- при наличии технической возможности: запуск релевантных тестов или ручную сценарную проверку.

## Правила перехода к следующей задаче

Автоматический переход к следующей задаче допускается только если одновременно выполнены все условия:

- критерии приёмки текущей задачи выполнены;
- в затронутых зонах не найдено новых проблем;
- проверка не выявила блокирующих дефектов;
- нет незакрытых замечаний, которые делают результат нестабильным.

## Если проверка не пройдена

Нужно сформировать короткий отчёт по задаче. Отчёт должен содержать:

- номер и название задачи;
- что было реализовано;
- что именно не прошло проверку;
- где проявляется проблема;
- является ли проблема блокирующей;
- предложение, как закрыть блок.

## Формат предложения по закрытию блока

Если задача не проходит проверку, нужно предложить один из следующих вариантов:

- точечная доработка в рамках текущей задачи;
- выделение отдельной зависимой задачи;
- откат части решения и выбор более безопасного варианта;
- перенос задачи в отдельный технический патч, если проблема не мешает текущему потоку.

## Работа с новыми проблемами

Если в процессе выполнения задачи обнаружена новая проблема, которая не исправляется в этом же патче, её нужно записать в [engineering_notes.md](/d:/Cursor/engineering_notes.md).

## Итоговая цель

Пул задач должен проходиться как непрерывная последовательность:

`реализация -> проверка -> либо автоматический переход дальше, либо отчёт и закрытие блока`

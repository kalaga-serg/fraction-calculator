# Спецификация калькулятора

Статус: черновик v0.1 от 2026-04-07

## 1. Цель

Калькулятор должен точно считать выражения с рациональными числами и показывать результат в двух представлениях:

- как сокращённую дробь;
- как десятичное число, пригодное для дальнейших вычислений.

## 2. Поддерживаемый ввод

На текущем этапе фиксируем такой поддерживаемый формат:

- целые числа: `2`, `15`, `0`;
- обычные дроби: `1/2`, `7/8`;
- десятичные числа: `1.5`, `0.25`;
- операции: `+`, `-`, `*`, `/`;
- скобки: `(` и `)`;
- пробелы внутри выражения допускаются и игнорируются;
- запятая во вводе в веб-версии нормализуется в точку.

## 3. Правила вычисления

- Вычисления должны вестись через рациональное представление `n/d`, а не через `float`/`Number` как основной источник истины.
- Каждая промежуточная дробь должна сокращаться.
- Деление на ноль должно завершаться понятной ошибкой.
- Приоритет операций стандартный: `*` и `/` выше `+` и `-`.
- Скобки должны менять порядок вычисления.

## 4. Правила отображения результата

### 4.1. Дробный вид

- Если знаменатель равен `1`, показывается только целое число.
- Иначе показывается сокращённая дробь в виде `n/d`.

### 4.2. Десятичный вид

- Если у результата меньше 15 знаков после запятой, нужно показывать только фактическое количество знаков.
- Если у результата больше 15 знаков после запятой, нужно показывать первые 15 знаков после запятой.
- Нельзя дописывать лишние нули только ради фиксированной длины.
- Нельзя получать десятичное представление через двоичное округление `float`/`Number.toFixed`, если это может добавить несуществующие цифры.
- Для конечных десятичных дробей результат должен быть точным.
- Для бесконечных десятичных дробей на текущем этапе используется усечение до 15 знаков после запятой.

Примеры:

- `1/8` -> `0.125`
- `1/3` -> `0.333333333333333`
- `1/40` -> `0.025`

## 5. Интерфейс

- После успешного вычисления пользователь видит дробный и десятичный результат.
- Кнопка повторного использования результата должна подставлять в поле ровно то значение, которое отображено в десятичном виде.
- История должна хранить последние 10 вычислений.

## 6. Подтверждённые расхождения с желаемым поведением

- В проекте нет единой спецификации, из-за чего веб- и Python-версии уже расходятся по возможностям.
- Поддержка отрицательных чисел и унарного минуса сейчас работает некорректно.
- В веб-версии строка с дробным результатом скрывается после первого успешного расчёта, это выглядит как спорное поведение и требует решения в следующей версии спецификации.
- В проекте нет автотестов на парсер, вычисления и форматирование результата.

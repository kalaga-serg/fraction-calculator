## Task 7. Спроектировать UX-решение для длинных выражений

Тип: `design`

Описание:
Подготовить решение для сценария, где пользователь вводит числа до 20 знаков и длинные выражения из 8-10 таких чисел. Рекомендуемое направление: перейти от однострочного поля к многострочному полю ввода с автоувеличением по высоте до разумного лимита и внутренней прокруткой после него.

Критерии приёмки:

- выбрано и зафиксировано одно UX-решение;
- пользователь может видеть и редактировать длинное выражение без визуального обрезания;
- решение подходит для десктопа и мобильного сценария;
- сценарий расчёта кнопкой сохраняется.

Зоны изменений:

- UX ввода
- мобильный и десктопный сценарии
- поведение длинных выражений в интерфейсе

## Task 8. Реализовать новый ввод для длинных выражений

Тип: `development`

Описание:
Внедрить утверждённое UX-решение для длинных выражений. Базовый ожидаемый вариант: многострочное поле ввода с автоувеличением по высоте, хорошей читаемостью длинных чисел и сохранением текущего сценария расчёта.

Критерии приёмки:

- поле принимает длинные выражения без визуального обрезания;
- ввод удобен на десктопе и мобильных;
- расчёт, фокус и подстановка результата работают корректно;
- длинные выражения из нескольких больших чисел можно просматривать и редактировать без поломки интерфейса.

Зоны изменений:

- [index.html](/d:/Cursor/index.html)
- разметка поля ввода
- стили поля ввода
- JS-логика расчёта и фокуса
